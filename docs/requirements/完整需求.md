# 模块0：用户登录与身份认证

**一、功能需求1：多角色账号登录**

**（一）需求定义**

1.  **需求 ID：**AUTH-001

2.  **需求名称：**多角色用户账号登录

3.  **需求描述：**

系统支持不同身份用户通过账号密码登录，包括学生用户与教职工用户（含教师，HR）。用户登录时仅需选择"学生"或"教师"身份。登录界面密码输入框右侧设有“眼睛”图标，点击可切换密码明文/密文显示。系统在用户登录成功后，根据账号角色自动进入对应功能主页，并加载相应的数据权限与界面内容。

4.  **前置条件：**

<!-- -->

(1) 用户已完成账号注册或由管理员分配账号；

(2) 系统服务正常运行；

(3) 用户账号具备合法角色身份。

<!-- -->

5.  **输入：**

    (1) 用户名

    (2) 用户账号（学号 / 工号 / 邮箱）

    (3) 登录密码

    (4) 用户角色（学生 / 教师）

**（二）处理逻辑：**

1.  用户输入账号、密码并选择登录角色（学生 或 教师）；

2.  系统校验输入项是否为空；

3.  查询数据库用户信息；

4.  验证账号密码是否正确（演示账号：`student/123`、`staff/123`）；

5.  校验用户账号是否具备所选角色权限；

6.  登录成功后创建用户会话；

7.  根据实际角色跳转对应系统页面：

8.  学生用户：进入个人成长管理主页

9.  教师用户：进入教师工作台主页（参照学生首页布局）

10. 若验证失败，则返回错误提示。

**（三）输出 / 预期结果：**

1.  登录成功：

<!-- -->

(1) 成功进入对应角色主页；

(2) 加载角色权限菜单；

(3) 保持登录状态。

<!-- -->

2.  登录失败：

<!-- -->

(1) 提示账号或密码错误；

(2) 或提示角色权限不匹配。

**（四）验收标准：**

1.  学生用户及教师用户均可正常登录

2.  不同角色登录后进入不同主页；

3.  无权限角色不可进入其他角色页面；

4.  错误账号密码无法登录；

5.  密码可见切换功能正常；

6.  登录过程响应正常，无异常报错。

**二、功能需求2：角色权限控制**

**（一）需求定义**

1.  **需求 ID**：AUTH-002

2.  **需求名称**：用户角色权限分配

3.  **需求描述**：

> 系统根据登录角色自动控制用户可访问的功能模块与数据范围，确保不同身份用户仅能使用对应权限内容。

4.  **前置条件**：

> 用户已成功登录系统。

5.  **输入**：

> 当前登录用户角色信息。

**（二）处理逻辑：**

1.  学生用户权限：

<!-- -->

(1) 管理个人课程成绩

(2) 管理项目经历

(3) 生成简历

(4) 查看大众对比分析

(5) 进行视图切换（学生视图/展示视图）

<!-- -->

2.  教师用户权限：

<!-- -->

(1) 查看教师工作台首页

(2) 查看学生展示视图（多学生简历卡片网格）

(3) 查看对比分析页面（含基准数据导入与多维度对比）

(4) 新增学生档案（输入姓名、学号）

(5) 不可编辑或修改任何学生数据。

**（三）输出 / 预期结果：**

不同角色仅展示对应菜单与页面内容。

**（四）验收标准：**

1.  学生无法进入教师管理页；

2.  教师无法编辑学生数据；

3.  权限切换准确无误；

4.  页面菜单与角色一致。

## 模块1：记录所学课程，记录成绩以及TPA计算 (核心功能：用户可以清晰、便捷地录入和管理课程信息，查询成绩明细，并自动准确地进行GPA等相关统计。)

**（新增）七、功能需求7：各学期GPA趋势图表**

**（一）需求定义**

1.  **需求 ID：**CGM=07

2.  **需求名称：**各学期GPA趋势可视化

3.  **需求描述**：

> 在课程管理页面底部，系统应以柱状图形式展示各学期GPA（或平均成绩）的变化趋势。X轴为学期名称（如“2024春”），Y轴为GPA数值（或百分制平均分）。图表随课程数据变动自动更新，切换评分机制时图表数据与坐标轴标签同步刷新。

4.  **前置条件**：课程列表中至少有一门课程数据。

5.  **输入**：当前用户全部课程数据及所选评分机制（4.0绩点制/百分制）。

**（二）处理逻辑**：

1. 聚合各学期课程数据，计算每学期加权平均绩点或平均分；
2. 调用图表库（Chart.js）渲染柱状图；
3. 当课程数据发生增、删、改或评分机制切换时，自动销毁旧图表并重新绘制。

**（三）输出 / 预期结果**：

柱状图清晰展示各学期GPA/平均分变化，数据准确，切换评分机制时图表同步更新。

**（四）验收标准**：

1. 图表随课程数据变动实时刷新；
2. 切换评分机制后，图表标题、数值与坐标轴正确；
3. 无数据时图表区域显示占位提示。

**（一）需求定义**

1.  **需求 ID：**RESUME-001

2.  **需求名称：**自动生成简历

3.  **需求描述：**

> 系统整合用户的基本信息、课程与成绩信息、GPA、成就与项目经历、技能特长等内容，按照预设的专业求职简历模板自动生成完整、规范的PDF或Word格式简历。简历采用清晰的双栏或单栏布局，包含个人信息、教育背景、专业技能、实习经历、项目经历、荣誉奖项、语言/证书等标准模块，并支持用户选择不同模板风格。生成后提供页面内嵌简版预览、弹窗完整预览及下载功能。

4.  **前置条件：**

<!-- -->

(1) 用户已登录系统

(2) 用户已完善基本信息（姓名、学校、专业、联系方式等）

(3) 用户已录入课程、成绩、成就或项目等相关数据

<!-- -->

5.  **输入：**

<!-- -->

(1) 基本信息：姓名、性别、电话、邮箱、求职意向、个人照片（可选）；

## 模块5：大众数据对比分析 (创新功能：个人数据与同群体数据多维度对比)

**一、功能需求1：大众基准数据导入**

**（一）需求定义**

1.  **需求 ID：**COMPARE-001

2.  **需求名称：**大众基准数据导入

3.  **需求描述：**

4.  **前置条件：**

<!-- -->

(1) 用户已登录系统，且已录入个人成长相关数据；

(2) 系统内已成功导入所选对比群体对应的有效基准数据。

<!-- -->

5.  **输入：**

<!-- -->

(1) 目标对比群体选择（下拉选项：同校/同专业/同年级/同升学方向/同就业方向）、对比维度选择（多选：GPA排名、课程修读进度、竞赛经历、实习经历、获奖情况、技能证书持有情况）；

(2) 对比时间范围、数据展示粒度（按学期/按学年/按全大学周期）

**（二）处理逻辑：**

1.  **前置数据校验：**校验用户是否已录入个人数据，校验所选群体是否存在可用的基准数据，无对应数据则提示用户补充录入或更换对比群体；

2.  **双端数据提取：**根据用户选择的对比维度，提取用户个人数据，并匹配所选群体对应维度的基准统计数据（均值、中位数、前20%分位值、前50%分位值等指标）；

3.  **分类对比计算：**

<!-- -->

(1) [数值型数据（如GPA、课程成绩）]{.underline}：计算个人数值在群体中的分位排名、与群体均值/中位数/优秀线的差值；

(2) [枚举型数据（如竞赛/实习经历、证书持有）]{.underline}：计算个人经历覆盖度与群体占比的对标情况，展示同群体拥有同类经历的人数占比；

<!-- -->

4.  **结果整合：**按用户选择的维度分类整合对比结果，生成差异分析（如“你的GPA处于同专业前20%，超过80%的同年级同学”）；

5.  **可视化展示：**在对比结论下方以柱状图展示个人数值与群体均值的直观对比；

6.  **异常容错处理：**对比过程中若出现单维度数据缺失，自动跳过该维度并给出提示，不影响其他维度的正常对比计算。

**（三）输出 / 预期结果：**

1.  **对比成功：**生成可视化的对比分析结果（含文字结论与柱状图），分维度展示个人数据与群体基准的对标情况，附带成长定位结论；

2.  **对比失败：**给出失败原因（如无个人数据、无对应基准数据）。

**（四）验收标准：**

1.  无个人数据或无匹配基准数据时，不触发对比；

2.  对比计算逻辑准确，分位值、差值、占比等数据计算无误，与基准数据匹配一致；

3.  多维度对比可正常勾选组合，各维度结果独立展示，无数据错乱、交叉错误；

4.  自动生成的对比结论与数据结果完全一致；

5.  柱状图正确渲染个人与群体对比数据；

6.  页面响应流畅，对比操作无卡顿，主流浏览器兼容正常。

## 模块7：UI视觉设计与交互规范（新增）

### 7.1 设计原则

- **温暖极简主义**：低视觉负担、高信息效率、阅读沉浸感。
- **一致性**：所有页面遵循统一的色彩、字体、圆角、阴影、间距规范。
- **克制动效**：仅使用悬停、聚焦等必要微交互。

### 7.2 色彩系统

| 用途 | 色值 | 名称 |
| :--- | :--- | :--- |
| 页面背景 | `#F5F4ED` | Parchment |
| 卡片/容器背景 | `#FAF9F5` | Ivory |
| 深色强调背景 | `#30302E` | Dark Surface |
| 主文字 | `#141413` | Near Black |
| 辅助文字 | `#5E5D59` | Olive Gray |
| 三级文字/占位符 | `#87867F` | Stone Gray |
| 边框分割线 | `#F0EEE6` | Border Cream |
| 品牌强调色 | `#C96442` | Terracotta |
| 悬停/聚焦环 | `#D1CFC5` | Ring Warm |
| 错误提示 | `#B53333` | Error |

### 7.3 字体规范

| 层级 | 字体族 | 字号/字重 | 用途 |
| :--- | :--- | :--- | :--- |
| 页面主标题 | Georgia, serif | 36-48px / 500 | 各页面大标题 |
| 卡片标题 | Georgia, serif | 20-24px / 500 | 卡片内部标题 |
| 正文 | Arial, system-ui, sans-serif | 14-16px / 400 | 功能性文字、按钮、表单 |
| 辅助说明 | Arial, sans-serif | 12-13px / 400 | 描述文字、时间戳 |
| 标签/代码 | SFMono, monospace | 13px / 400 | 数据标注、绩点数值 |

### 7.4 布局与间距

- **基准网格**：8px
- **侧边栏宽度**：280px
- **内容区内边距**：32px 40px
- **卡片内边距**：24-28px
- **卡片间距**：24px
- **圆角**：卡片 20-24px，按钮 12-40px，输入框 12-18px
- **阴影**：卡片 `0 4px 24px rgba(0,0,0,0.04)`；悬浮卡片可加深至 `0 8px 30px rgba(0,0,0,0.08)`

### 7.5 核心组件规范

#### 7.5.1 左侧导航菜单
- 背景：半透明羊皮纸 + 毛玻璃模糊（`backdrop-filter: blur(12px)`）
- 菜单项高度：48px，内边距 14px 18px
- 选中态：纯白背景 + 陶土红文字 + 环形阴影 `0 0 0 1px #D1CFC5`
- 图标：线性图标，颜色跟随文字

#### 7.5.2 按钮
| 类型 | 背景 | 文字颜色 | 边框/阴影 | 悬停 |
| :--- | :--- | :--- | :--- | :--- |
| 品牌主按钮 | `#C96442` | `#FAF9F5` | `0 0 0 1px #C96442` | 背景加深至 `#B5583A`，轻微上浮 |
| 次级按钮 | `#E8E6DC` | `#4D4C48` | `0 0 0 1px #D1CFC5` | 背景加深至 `#D9D3C5` |
| 文字按钮 | 透明 | `#C96442` | 无 | 增加下划线 |

#### 7.5.3 表单输入框
- 背景纯白 `#FFFFFF`，边框 `#F0EEE6`，圆角 12px，内边距 14px 16px
- 聚焦：边框变为 `#C96442`，外圈 3px `rgba(201,100,66,0.12)` 光晕
- 错误状态：边框变为 `#B53333`

#### 7.5.4 卡片
- 标准卡片：象牙白背景 + 1px 奶油边框 + 20-24px 圆角 + 轻阴影
- 深色卡片（GPA展示区）：`#30302E` 背景，白色文字，无边框
- 首页双栏卡片需严格底部对齐

#### 7.5.5 数据可视化（柱状图）
- 采用 Chart.js 渲染
- 个人数据柱：`#C96442`（陶土红）
- 群体均值柱：`#D1CFC5`（暖灰）
- 坐标轴标签：`#5E5D59`，网格线：`#F0EEE6`

### 7.6 响应式策略
- 桌面端优先，最小宽度 1200px
- 移动端：侧边栏收缩为汉堡菜单（二期迭代）
