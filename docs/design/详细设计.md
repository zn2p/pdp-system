版本: 1.0
日期: 2026-04-25

摘要
本设计文档基于产品原型（学生/教师端仪表盘、课程管理、成就管理、简历生成与大众对比）描述后端技术选型、模块划分、主要数据模型与 API 接口定义。后端语言为 Python，数据库使用 SQLite。

- 附录：示例请求/响应与错误码

---

技术选型

- 开发语言: Python 3.10+
- Web 框架: FastAPI（异步、自动生成 OpenAPI、与 Pydantic 强集成）
- ASGI 服务器: Uvicorn（或 Gunicorn + Uvicorn workers）
- 数据库: SQLite（sqlite:///./pdp.db），满足轻量、单文件、易部署的需求
- ORM: SQLAlchemy（1.4+），配合 Pydantic/SQLModel 用于模型定义
- 数据迁移: Alembic（注意 SQLite 的一些迁移受限）
- 鉴权: JWT（PyJWT 或 python-jose），基于 role（student/staff）做权限控制
- 表单/文件处理: FastAPI 内置 `UploadFile`
- 异步任务（可选）: Celery / RQ（大文件导入/异步导出/复杂对比），开发阶段可使用同步实现
- 日志: Python logging / structlog
- 依赖管理: requirements.txt 或 poetry

设计理由：FastAPI 提高开发效率并自动生成文档，SQLite 便于本地演示与测试，SQLAlchemy 与 Pydantic 提供强校验与映射能力。

系统架构与模块划分

代码目录建议（位于 `app/`）

- `app/main.py` — 应用入口，创建 FastAPI 实例、注册路由与中间件
- `app/api/v1/` — 路由目录（auth.py, users.py, students.py, courses.py, achievements.py, resume.py, benchmark.py, files.py）
- `app/core/` — 配置（settings）、JWT 工具、依赖注入 helpers
- `app/db/` — 数据库会话（SessionLocal）、初始化与 Alembic 配置
- `app/models/` — SQLAlchemy ORM 模型
- `app/schemas/` — Pydantic 请求/响应模型
- `app/services/` — 业务逻辑（GPA 计算、对比算法、简历渲染）
- `app/tasks/` — 异步任务（Celery/RQ 任务）
- `app/utils/` — 文件处理、CSV/XLSX 解析、导出工具

模块职责概述

- auth：登录、JWT 签发与权限检查
- users：用户基本信息与角色管理
- students：学生资料聚合（课程、成就、简历数据）
- courses：课程 CRUD、绩点/GPA 计算接口
- achievements：成就/证书/项目 CRUD 与附件管理
- benchmark：基准数据导入（CSV/XLSX）与对比分析入口
- resume：简历 HTML 渲染与导出（PDF/Word）

数据库设计（主要表）

说明：使用 SQLite，表主键均为自增整型，包含常用 `created_at`/`updated_at` 字段（省略在此处）。

1. `users`
- `id` INTEGER PK
- `username` TEXT UNIQUE
- `password_hash` TEXT
- `display_name` TEXT
- `role` TEXT ('student'|'staff'|'admin')
- `email` TEXT

2. `students`
- `id` INTEGER PK
- `user_id` INTEGER FK -> users.id
- `student_id` TEXT UNIQUE
- `school` TEXT
- `major` TEXT
- `grad_year` TEXT
- `phone` TEXT
- `photo_path` TEXT

3. `courses`
- `id` INTEGER PK
- `student_id` INTEGER FK -> students.id
- `name` TEXT
- `code` TEXT
- `semester` TEXT
- `credit` REAL
- `grade` REAL -- 百分制
- `teacher` TEXT
- `rank` TEXT
- `note` TEXT

4. `achievements`
- `id` INTEGER PK
- `student_id` INTEGER FK
- `name`, `type`, `date`, `org`, `level`, `description`, `tags`, `attachment_path`

5. `benchmarks`
- `id` INTEGER PK
- `group_name` TEXT
- `time_range` TEXT
- `source_filename` TEXT
- `metadata` JSON / TEXT
- `imported_count` INTEGER

6. `benchmark_records`
- `id` INTEGER PK
- `benchmark_id` FK
- `student_identifier` TEXT
- `gpa` REAL
- `fields` JSON / TEXT

7. `files`
- `id` INTEGER PK
- `owner_id` INTEGER
- `path` TEXT
- `filename` TEXT
- `mime` TEXT
- `size` INTEGER

索引建议：为 `users.username`, `students.student_id`, `courses(student_id, semester)` 添加索引。

注意：SQLite 写并发受限，如需高并发写操作，建议迁移到 PostgreSQL。

API 接口定义（按模块）

全局约定：

- Base URL: `/api/v1`
- 鉴权: 除登录注册外均需 `Authorization: Bearer <token>`
- 返回格式: JSON `{ "code": 0, "message": "OK", "data": ... }`
- 常见 HTTP 状态码：200 成功，201 创建成功，400 参数错误，401 未鉴权，403 无权限，404 未找到，500 服务异常

1) Auth

- POST `/api/v1/auth/login`
  - 描述: 登录并返回 access token
  - 请求体: `{ "username": "", "password": "" }`
  - 成功返回: `{ "access_token": "..", "token_type": "bearer", "expires_in": 3600, "user": {"id":1,"display_name":"...","role":"student"} }`

- POST `/api/v1/auth/refresh`（可选）

2) Users

- GET `/api/v1/users/me`
  - 描述: 当前用户信息

3) Students

- GET `/api/v1/students`
  - 描述: 列表（教师查看其学生或管理员查看全部）
  - 参数: `?page=&limit=&q=`

- POST `/api/v1/students`
  - 创建学生

- GET `/api/v1/students/{student_id}`
  - 获取学生详情（包括 courses & achievements）

4) Courses

- GET `/api/v1/students/{student_id}/courses`
  - 列表

- POST `/api/v1/students/{student_id}/courses`
  - 创建课程 `{ name, semester, code, credit, grade, teacher, note }`

- PUT `/api/v1/students/{student_id}/courses/{course_id}`
  - 更新

- DELETE `/api/v1/students/{student_id}/courses/{course_id}`

- GET `/api/v1/students/{student_id}/gpa`
  - 返回累计/分学期 GPA
  - 示例返回: `{ "total_gpa":3.65, "total_credits":32, "by_semester": [{"semester":"2024春","gpa":3.5}] }`

5) Achievements

- GET `/api/v1/students/{student_id}/achievements`
- POST `/api/v1/students/{student_id}/achievements` `{ name,type,date,org,level,description,tags,attachment_id? }`
- PUT `/api/v1/students/{student_id}/achievements/{achieve_id}`
- DELETE `/api/v1/students/{student_id}/achievements/{achieve_id}`

6) Files

- POST `/api/v1/files/upload` (multipart/form-data)
  - 返回 `{ file_id, url, filename, mime, size }`

- GET `/api/v1/files/{file_id}/download`

7) Benchmark 导入

- POST `/api/v1/benchmark/import` (multipart)
  - 参数: `file, group_name, time_range`
  - 返回：导入摘要或任务 id `{ task_id, success_count, fail_count }`

- GET `/api/v1/benchmark/{id}/status`

8) Compare 分析

- POST `/api/v1/compare/run`
  - 请求: `{ student_id, group, dims:['gpa','course'], time_range }`
  - 返回: `{ summary, details:{ gpa:{...}, course:{...} }, charts: { ... } }` 或任务 id

9) Resume

- POST `/api/v1/students/{student_id}/resume/preview` `{ template }` 返回 HTML 或 Base64
- POST `/api/v1/students/{student_id}/resume/export` `{ format:'pdf'|'word', template }` 返回下载链接或任务 id

POST `/api/v1/students/2024001/courses`

请求体

{
  "name": "数据库",
  "semester": "2024春",
  "code": "CS301",
  "credit": 3,

```
{
  "code": 4001,
  "message": "学分必须为正数"
}
```

鉴权与安全

- 使用 JWT（短期 access token + 可选 refresh token）
- 密码使用 bcrypt/argon2 哈希
- 路由依赖注入（FastAPI Depends）进行角色校验
- 文件上传限制类型与大小；下载需鉴权
- Pydantic 做输入校验，防止注入/异常数据

部署与运维建议

- 开发/演示：SQLite + Uvicorn 即可
- 生产：推荐 PostgreSQL，使用 Nginx + Uvicorn/Gunicorn，添加 TLS
- 备份：定期备份 SQLite 文件，上传文件目录
- 监控：日志 + Sentry，暴露 /metrics 可供 Prometheus 抓取

附录：错误码范围

- 0 成功
- 1000~1999 鉴权/权限相关
- 2000~2999 参数/校验错误
- 3000~3999 业务错误
- 5000 服务器内部错误
